📖 核心术语深度解析

在代理和网络技术领域，有许多专业术语和"行业黑话"。本章将详细解释这些术语，帮助你更好地理解相关技术讨论。

服务器角色术语

落地机 (Exit Node)

定义

落地机是指最终访问目标网站的服务器，它的 IP 地址会暴露给目标网站。在代理链路中，落地机是流量的"出口"。

为什么叫"落地"？

因为它是流量最终"落地"、真正访问互联网的地方。目标网站看到的 IP 地址就是落地机的 IP。

关键特点

  • ✅ 需要在目标地区（如美国、日本、香港）
  • ✅ IP 质量很重要（避免被封禁）
  • ✅ 带宽要足够（影响实际速度）
  • ⚠️ 容易被目标网站检测和封禁

线路机 / 中转机 (Relay Node)

定义

线路机或中转机是位于客户端和落地机之间的中间服务器，用于转发流量。它可以隐藏落地机的真实 IP，也可以优化网络路由。

为什么需要中转？

  • 🚀 线路优化 - 绕过国际网络拥堵，使用优质线路（如 CN2）
  • 🛡️ 隐藏落地机 - 防止落地机 IP 暴露给审查系统
  • ⚡ 负载均衡 - 一个中转可以连接多个落地机
  • 🔐 安全隔离 - 即使中转被封，落地机仍然安全

典型架构

用户 → 国内中转 → 香港中转 → 美国落地机 → 目标网站
      (CN2 线路)  (IPLC 专线)    (原生IP)

中转 (Relay / Forward)

中转是一种技术手段，通过中间服务器转发流量。常见的中转方式包括：

  • 端口转发 - 使用 iptables、socat 等工具在 IP 层转发
  • 应用层中转 - 使用 Nginx、HAProxy 等在应用层转发
  • 隧道中转 - 使用 WireGuard、GRE 隧道等
  • CDN 中转 - 使用 Cloudflare 等 CDN 服务转发

网络与 IP 类型

家宽 (Residential IP)

特点 说明
定义 家庭宽带提供的 IP 地址，也称住宅 IP
IP 段特征 属于 ISP 分配给家庭用户的 IP 段
信誉度 ⭐⭐⭐⭐⭐ 极高，不易被封禁
稳定性 ⭐⭐⭐ 中等，可能动态变化
带宽 上行通常较低（1-10 Mbps）
价格 💰💰💰 昂贵（$20-100/月）
适用场景 需要高信誉度的场景（爬虫、账号养号等）

机房 IP / 数据中心 IP (Data Center IP)

特点 说明
定义 数据中心机房的商业 IP 地址
IP 段特征 属于 IDC、云服务商的 IP 段
信誉度 ⭐⭐⭐ 中等，容易被识别
稳定性 ⭐⭐⭐⭐⭐ 非常稳定
带宽 通常很高（100Mbps - 10Gbps）
价格 💰 便宜（$5-20/月）
适用场景 个人代理、一般性科学上网

原生 IP vs 广播 IP

原生 IP (Native IP)

IP 的注册地和实际服务器所在地一致的 IP。例如日本机房使用日本注册的 IP。

优势：可以解锁地区限制的流媒体服务（如 Netflix、HBO）

广播 IP (Broadcast IP)

IP 注册地和实际服务器所在地不一致。例如美国机房使用香港注册的 IP。

劣势：可能无法解锁流媒体，被认为是代理

线路类型术语

中国电信的下一代承载网，是目前质量最好的国际线路之一。

  • ✅ 延迟最低，速度最快
  • ✅ 晚高峰不拥堵
  • ⭐⭐⭐⭐⭐ 最优质
 💰💰💰 贵

去程与回程

去程 (Outbound)

从你的设备到代理服务器的路径。影响你发送请求的速度。

回程 (Inbound / Return)

从代理服务器返回到你设备的路径。影响接收数据的速度。

为什么回程更重要？

因为下载数据量远大于上传（浏览网页、看视频主要是下载），所以回程质量对用户体验影响更大。

IPLC (International Private Leased Circuit)

定义

国际专线，是点对点的物理专线连接，不经过公网。

特点

  • ⚡ 极低延迟 - 物理专线，延迟最低
  • 🚀 极高速度 - 不受公网拥堵影响
  • 🛡️ 不过墙 - 不经过 GFW，几乎不会被封
  • 💰 极贵 - 价格是普通 VPS 的 5-10 倍

价格参考

通常 $50-200/月，按流量或带宽计费。

BGP (Border Gateway Protocol)

BGP 线路是指服务器同时接入多条线路（电信、联通、移动等），通过 BGP 协议自动选择最优路径。

  • ✅ 多线接入，各运营商用户都能获得好速度
  • ✅ 自动故障切换，稳定性高
  • ⚠️ 价格较贵

技术术语详解

TLS

TLS（Transport Layer Security）是 HTTPS 的安全基础，用于在客户端和服务器之间建立加密通道，保证机密性与完整性。

  • ✅ 你日常访问 https:// 网站，基本都在使用 TLS
  • ✅ 代理协议常用 TLS 来“看起来像正常 HTTPS”，提升隐蔽性（如 Trojan、WS+TLS）
  • ⚠️ 现代审查通常不只看“有没有 TLS”，还会看 TLS 指纹、SNI、握手特征与流量形态

与代理相关的几个关键点

  • TLS 指纹：客户端握手参数组合会形成可识别特征（不同库/设备差异很大）
  • SNI：明文指示要访问的域名（TLS 1.3 仍可能暴露），常用于策略判断
  • 证书：自签名/异常证书更容易引起怀疑；使用正规证书更“像正常站点”

Sing-box

一个通用代理平台，类似 V2Ray，但更现代化，支持多种协议。

  • ✅ 支持 VLESS、Trojan、Hysteria2、TUIC 等新协议
  • ✅ 性能优秀，资源占用低
  • ✅ 配置简洁，易于维护
  • ⚠️ 相对较新，社区相对小

混淆 (Obfuscation)

通过技术手段伪装流量特征，使其看起来像普通流量（如 HTTPS、HTTP）。

常见混淆方式

  • TLS 混淆 - 伪装成 HTTPS 流量
  • WebSocket 混淆 - 伪装成 WebSocket 连接
  • HTTP 伪装 - 伪装成普通 HTTP 流量
  • CDN 伪装 - 通过 Cloudflare 等 CDN 转发

沙箱 (Sandbox)

隔离的运行环境，用于安全地测试和运行不明程序。

  • 🔒 隔离系统环境，防止恶意代码破坏
  • 🧪 测试不信任的脚本和程序
  • 💡 常用技术：Docker、虚拟机、chroot

DPI (Deep Packet Inspection)

深度包检测，一种网络流量分析技术，能够检查数据包的内容和特征。

  • 🔍 GFW 使用 DPI 识别和封锁代理流量
  • 🛡️ 现代协议通过混淆对抗 DPI
  • ⚠️ 简单的加密（如老版本 SS）容易被 DPI 识别

QoS (Quality of Service)

服务质量控制，ISP 可以通过 QoS 对特定流量进行限速或优先级控制。

  • ⚠️ 运营商可能对国际流量进行 QoS 限速
  • ⚠️ 识别到代理流量后可能降低优先级
  • ✅ 使用伪装和混淆可以绕过 QoS

常见场景术语

翻墙 / 科学上网

使用代理技术突破网络审查，访问被屏蔽的网站和服务。这是代理技术最常见的应用场景。

落地 IP 被墙

指代理服务器的 IP 地址被防火墙封锁，无法正常连接。

  • ⚠️ 表现：连接超时、丢包严重
  • 🔧 解决：更换服务器 IP 或端口
  • 🛡️ 预防：使用高隐蔽性协议、避免大流量

端口被墙

特定端口被防火墙封锁，但 IP 地址仍然可用。

  • ✅ 表现：某些端口无法连接，但其他端口正常
  • 🔧 解决：更换端口（推荐使用 443、80 等常见端口）

机场

提供商业代理服务的平台，用户购买订阅后获得代理节点。得名于 "Airport"（Shadowsocks 客户端名称）。

  • ✅ 优点：开箱即用，无需自己搭建
  • ⚠️ 缺点：隐私风险、可能跑路、价格较贵

性能相关术语

延迟 (Latency / Ping)

数据从客户端到服务器的往返时间，单位通常是毫秒（ms）。

  • ⚡ 优秀：< 50ms（香港、日本）
  • ✅ 良好：50-100ms（新加坡、韩国）
  • ⚠️ 一般：100-200ms（美国西海岸）
  • ❌ 较差：> 200ms（美国东海岸、欧洲）

带宽 (Bandwidth)

网络连接的最大数据传输速率，单位通常是 Mbps 或 Gbps。

  • 📊 VPS 通常提供 100Mbps - 1Gbps 带宽
  • ⚠️ 实际速度取决于线路质量，不是带宽越大越好

流量 (Traffic / Transfer)

数据传输的总量，单位通常是 GB 或 TB。

  • 📊 个人使用：50-200GB/月
  • 📊 重度使用（看视频）：500GB-1TB/月
  • 💡 VPS 通常提供 500GB - 数 TB 流量
