🔍 GFW 检测原理

了解 GFW（Great Firewall）的检测机制，有助于我们选择更隐蔽的代理方案。

GFW 的主要检测技术

1. IP 黑名单

工作原理

维护一个被封禁的 IP 地址列表，直接阻断与这些 IP 的连接。

触发条件

  • 被识别为代理服务器
  • 频繁访问敏感网站
  • 异常流量模式
  • 被举报

对抗方法

  • ✅ 定期更换 IP
  • ✅ 使用住宅 IP
  • ✅ 避免异常流量

2. DPI 深度包检测

工作原理

检查数据包的内容和特征，识别代理协议的流量模式。

可检测的特征

  • 协议指纹（如 Shadowsocks 的特定模式）
  • 加密流量特征
  • TLS 握手异常
  • 数据包时序特征

对抗方法

  • ✅ 使用 Reality（完美伪装 TLS）
  • ✅ WebSocket + TLS（伪装成正常 HTTPS）
  • ✅ 流量混淆
  • ✅ 使用标准 TLS 证书

3. 主动探测

工作原理

GFW 主动连接疑似代理服务器，尝试发送代理协议的握手包，如果服务器响应则确认是代理。

触发条件

  • 检测到可疑的 TLS 握手
  • 服务器在常见代理端口监听
  • 流量模式异常

对抗方法

  • ✅ VLESS Reality（无法被探测）
  • ✅ 设置回落机制（探测时返回正常网站）
  • ✅ 使用认证机制
  • ✅ 避免使用自签名证书

4. DNS 污染

工作原理

返回错误的 DNS 解析结果，将域名指向错误的 IP 地址。

对抗方法

  • ✅ 使用加密 DNS（DoH、DoT）
  • ✅ 使用可信的国外 DNS（1.1.1.1、8.8.8.8）
  • ✅ 在代理内解析 DNS

5. QoS 限速

工作原理

识别国际流量或可疑流量后降低优先级，进行限速。

表现

  • 晚高峰严重降速
  • YouTube 等视频卡顿
  • 下载速度异常慢

对抗方法

  • ✅ 使用优质线路（CN2、IPLC）
  • ✅ 流量伪装成正常 HTTPS
  • ✅ 使用 CDN 中转

流量特征分析

容易被识别的特征

  • ❌ 固定的数据包大小
  • ❌ 规律的时间间隔
  • ❌ 异常的 TLS 握手
  • ❌ 自签名或无效证书
  • ❌ 非标准的端口使用
  • ❌ 持续的大流量传输

隐蔽的流量特征

  • ✅ 模拟真实浏览器的 TLS 指纹
  • ✅ 使用合法域名和证书
  • ✅ 标准端口（443、80）
  • ✅ 流量大小和时序随机化
  • ✅ 模拟正常网站访问模式

最佳实践建议

协议选择

环境 推荐方案 原因
严格封锁 VLESS Reality 完美伪装，无法识别
一般环境 Trojan + 前置网站 高隐蔽性，成本适中
追求速度 Hysteria2 + 混淆 速度快，需要频繁换端口

配置要点

  1. 使用标准端口 - 443 (HTTPS) 或 80 (HTTP)
  2. 配置正规证书 - Let's Encrypt 免费证书
  3. 前置伪装网站 - 访问根目录显示正常网站
  4. 启用回落机制 - 探测时返回正常内容
  5. 避免大流量 - 不要用于 BT 下载
  6. 控制连接数 - 单用户 3-5 个连接

检测工具

测试 IP 是否被墙

# 从国内网络 ping
ping your-vps-ip

# 使用在线工具
https://www.vps234.com/ipchecker/
https://tools.ipip.net/ping.php

测试端口是否被墙

# 测试端口连通性
telnet your-vps-ip 443

# 使用 tcping（Windows）
tcping your-vps-ip 443
